Unit 1: Fundamentals (Q1 - Q6)

Function Declaration vs Expression (Hoisting) Q1 - Q2

Function Declaration: Fully hoisted (both name and implementation) - can be called before it appears in code.

Function Expression: Variable is hoisted, but not the function assignment - must be defined before calling. 

// ‚úÖ DECLARATION - Works (fully hoisted)
greet(); // Output: "Hi"
function greet() { console.log("Hi"); }

// ‚ùå EXPRESSION - Error (variable hoisted, but undefined)
sayHi(); // TypeError: sayHi is not a function
var sayHi = function() { console.log("Hi"); }

// How hoisting actually works:
// Behind the scenes:
var sayHi;           // Variable declaration hoisted
sayHi();             // Called here (but sayHi is undefined)
sayHi = function() { // Assignment happens here
  console.log("Hi");
}

When to use:

  • Declaration: When you need hoisting, or for top-level utility functions
  • Expression: When you want to control when the function is available, or for callbacks

Temporal Dead Zone (TDZ) Q5

The time between entering a scope and the variable being declared where you cannot access the variable. 

console.log(x); // ReferenceError (TDZ)
let x = 5;
Scope Types Q6
  • Global Scope: Accessible everywhere
  • Function Scope: Accessible only within function (var)
  • Block Scope: Accessible within {} (let, const)
  • Lexical Scope: Inner functions can access outer function variables

Unit 2: Objects & Context (Q7 - Q9)

The 'this' Keyword Q7

The value of this depends on how a function is called:

  1. Default: Global object (window) or undefined (strict mode).
  2. Implicit: The object left of the dot (e.g., obj.method()).
  3. Explicit: Manually set via call, apply, or bind.
  4. Arrow Functions: Inherits this from the lexical scope. 
// Common Gotcha Solution (Arrow Function)
const person = {
  name: "David",
  greet: function() {
    setTimeout(() => {
      console.log(this.name); // "David" (lexical this)
    }, 1000);
  }
};
call, apply, and bind Q8
  • call(): Invokes immediately (args comma-separated)
  • apply(): Invokes immediately (args as array)
  • bind(): Returns new function with bound context
Arrow vs Regular Functions Q9

Arrow functions don't have their own this or arguments object and cannot be constructors.

Unit 4: Functional Programming (Q16 - Q21)

Pure vs Impure Functions Q16, Q17

Impure: Side effects (logs, global vars).

Pure: No side effects. Output depends only on input.

Closures Q18, Q19

A function that "remembers" its lexical scope even after the outer function has finished. 

function outer() {
  let count = 0;
  return function inner() {
    return ++count; // Remembers 'count'
  };
}
const counter = outer();
console.log(counter()); // 1
console.log(counter()); // 2

Practical Uses:

  • Data Privacy: Creating private variables.
  • Factories: Functions that generate specific behaviors.
  • Event Handlers: Preserving state in callbacks.
IIFE Q20, Q21

Immediately Invoked Function Expression. Used for Data Privacy.

Additional Important Interview Questions (Q26 - Q30)

Event Loop & Call Stack Q26

JavaScript is single-threaded but handles async operations through the event loop.

Components:

  • Call Stack: Stores function execution contexts
  • Web APIs: Browser APIs (setTimeout, fetch, DOM events)
  • Callback Queue: Stores callbacks ready to execute
  • Event Loop: Moves callbacks to call stack when empty 
console.log('1');

setTimeout(() => {
  console.log('2');
}, 0);

Promise.resolve().then(() => {
  console.log('3');
});

console.log('4');

// Output: 1, 4, 3, 2
// Why?
// - Synchronous code runs first: 1, 4
// - Promises (microtasks) before setTimeout (macrotasks): 3
// - setTimeout callback runs last: 2
== vs === Q27
  • == (Loose Equality): Compares values after type coercion
  • === (Strict Equality): Compares both value and type (no coercion) 
// Loose Equality (==)
5 == "5"           // true (coerced)
0 == false         // true
null == undefined  // true

// Strict Equality (===)
5 === "5"          // false (different types)
0 === false        // false
null === undefined // false

// Best Practice: Always use === unless you need coercion

Deep Copy vs Shallow Copy Q29

Shallow Copy (only copies first level):

const original = { name: "John", address: { city: "NYC" } };

// Spread operator or Object.assign()
const copy = { ...original };

// Problem: Nested objects still referenced
copy.address.city = "LA";
console.log(original.address.city); // "LA" (modified!)

Deep Copy (copies all levels):

// Method 1: JSON (simple objects only)
const deepCopy1 = JSON.parse(JSON.stringify(original));

// Method 2: Recursive function
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  const clone = Array.isArray(obj) ? [] : {};
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key]);
    }
  }
  return clone;
}

// Method 3: structuredClone (modern browsers)
const deepCopy3 = structuredClone(original);
Array Methods Comparison Q30 
const numbers = [1, 2, 3, 4, 5];

// forEach: Iterate (no return value)
numbers.forEach(num => console.log(num));
// Returns: undefined

// map: Transform (returns new array)
const doubled = numbers.map(num => num * 2);
// Returns: [2, 4, 6, 8, 10]

// filter: Keep elements that pass test
const evens = numbers.filter(num => num % 2 === 0);
// Returns: [2, 4]

// reduce: Reduce to single value
const sum = numbers.reduce((acc, num) => acc + num, 0);
// Returns: 15

// find: Return first match
const firstEven = numbers.find(num => num % 2 === 0);
// Returns: 2

// some: Check if ANY passes
const hasEven = numbers.some(num => num % 2 === 0);
// Returns: true

// every: Check if ALL pass
const allPositive = numbers.every(num => num > 0);
// Returns: true
