JavaScript Theory & Interview Questions

Comprehensive Guide: Ordered by Question Number

Unit 1: Fundamentals (Q1 - Q6)

Function Declaration vs Expression Q1 - Q2
  • Function Declaration: Hoisted. Can be called before definition.
  • Function Expression: Not hoisted. Must be defined before usage.
// 1. Declaration (Works)
greet(); 
function greet() { console.log("Hi"); }

// 2. Expression (Error)
sayHi(); 
var sayHi = function() { console.log("Hi"); }
Arrow Functions Q3

Concise syntax for functions.

let multiply = (a, b) => a * b;
var vs let vs const Q4
  • var: Function-scoped, hoisted, can be redeclared
  • let: Block-scoped, hoisted but not initialized (TDZ), cannot be redeclared
  • const: Block-scoped, must be initialized, cannot be reassigned
Temporal Dead Zone (TDZ) Q5

The time between entering a scope and the variable being declared where you cannot access the variable.

console.log(x); // ReferenceError (TDZ)
let x = 5;
Scope Types Q6
  • Global Scope: Accessible everywhere
  • Function Scope: Accessible only within function (var)
  • Block Scope: Accessible within {} (let, const)
  • Lexical Scope: Inner functions can access outer function variables

Unit 2: Objects & Context (Q7 - Q9)

The 'this' Keyword Q7

Refers to the object that is executing the current function.

  • Method → Object
  • Function → Global/Window
  • Arrow Function → Parent Scope
call, apply, and bind Q8
  • call(): Invokes immediately (args comma-separated)
  • apply(): Invokes immediately (args as array)
  • bind(): Returns new function with bound context
Arrow vs Regular Functions Q9

Arrow functions don't have their own this or arguments object and cannot be constructors.

Unit 3: Async & ES6 Features (Q10 - Q15)

Promises Q10

Object representing completion/failure of async operation. States: Pending, Fulfilled, Rejected.

Async/Await Q11

Syntactic sugar for Promises. await pauses execution until Promise resolves.

Destructuring Q12

Extract values from arrays/objects into variables.

const { name, age } = person;
const [first, second] = array;
Spread & Rest Operators Q13

Spread (...): Expands array/object.

Rest (...): Collects arguments into array.

Template Literals Q14

Backticks (`) allow string interpolation ${var} and multi-line strings.

Map vs forEach Q15
  • map: Returns a new array. Use for transformation.
  • forEach: Returns undefined. Use for side effects.

Unit 4: Functional Programming (Q16 - Q21)

Pure vs Impure Functions Q16, Q17

Impure: Side effects (logs, global vars).

Pure: No side effects. Output depends only on input.

Closures Q18, Q19

Function remembering its parent scope variables.

function outer() {
    let count = 0;
    return () => ++count;
}
IIFE Q20, Q21

Immediately Invoked Function Expression. Used for Data Privacy.

Unit 5: Practical & Advanced (Q22+)

Find vs Filter Q22

Find: Returns first match.

Filter: Returns array of all matches.

Some vs Every Q23

Some: True if ANY match.

Every: True if ALL match.

Coding: Private Counter Q24 
function counter (){
    let count = 0;
    return function(){ return ++count; }
}
Coding: Discount HOF Q25 
function discount(d) {
    return (price) => price - (price * d/100);
